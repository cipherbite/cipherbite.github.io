Shadow Tracker: Analyzing User Behavior in Tor and VPN

Shadow Tracker is a beginner-friendly cybersecurity project that explores how users interact with privacy tools like Tor and VPNs. By analyzing artifacts such as Tor’s browsing history and VPN connection logs, it reveals patterns like session times and IP changes, visualized beautifully in Grafana dashboards.

This project is perfect for learning digital forensics and data visualization. It’s relevant for cybercrime investigations and threat intelligence, showcasing skills in Python, InfluxDB, and Grafana.

Technologies Used

  • Python: Extracts and processes data from Tor and VPN artifacts for storage.
  • SQLite3: Reads browsing history from Tor’s places.sqlite database.
  • Volatility3: Analyzes system memory to verify Tor/VPN activity.
  • InfluxDB: Stores time-series data like session timestamps and IPs.
  • Grafana: Creates real-time dashboards to visualize user behavior.
  • Tor Browser: Generates browsing artifacts for analysis.
  • OpenVPN: Logs VPN connections for tracking.

These tools are widely used in cybersecurity and data engineering, making them great for learning and portfolio-building.

1. Setting Up the Environment

I set up a virtual machine with Ubuntu 20.04 to simulate user activities in a safe, isolated environment.

  • InfluxDB: Set up to store time-series data.
  • Grafana: Installed for dashboard creation.
  • Python: Used with sqlite3 and influxdb-client.

I launched Tor Browser and OpenVPN, then created a memory dump to capture system state.

2. Collecting Artifacts

I collected data from Tor and VPN to understand user behavior. Below is an example of parsing Tor’s browsing history: 


import sqlite3
from influxdb_client import InfluxDBClient, Point
from influxdb_client.client.write_api import WritePrecision

# Connect to Tor's places.sqlite
conn = sqlite3.connect("places.sqlite")
cursor = conn.cursor()
cursor.execute("SELECT url, visit_date FROM moz_places")
visits = cursor.fetchall()

# Send to InfluxDB
client = InfluxDBClient(url="http://localhost:8086", token="my-token", org="my-org")
write_api = client.write_api()
for url, timestamp in visits:
    point = Point("tor_visits").field("url", url).time(timestamp, WritePrecision.MS)
    write_api.write(bucket="tor_vpn_data", record=point)
conn.close()
  • Tor Browsing History: Parsed places.sqlite to extract URLs and timestamps of visited sites.
  • VPN Logs: Extracted connection times and IP addresses from OpenVPN logs.
  • Memory Check: Used Volatility3 to confirm Tor and VPN processes were running.

The parsed data was saved to InfluxDB for visualization.

3. Storing Data in InfluxDB

InfluxDB was configured to store time-series data, like session timestamps and IP connections. I created a database called tor_vpn_data and used Python to send parsed artifacts.

This step ensured data was ready for Grafana to visualize.

4. Visualizing with Grafana

Grafana dashboards brought the data to life, using queries like SELECT MEAN(visit_count) FROM tor_visits WHERE time >= now() - 1h GROUP BY time(5m). I created two main panels:

  • Session Timeline: A line chart showing when Tor sessions occurred.
  • IP Connections: A table listing VPN IPs and connection times.

These visualizations made it easy to spot trends, like frequent VPN IP changes.

5. Detecting Suspicious Activity

I used simple rules to flag suspicious behaviors, such as Tor sessions shorter than 1 minute or VPN connections with multiple IP changes in an hour. Below is an example of the detection logic: 


from influxdb_client import InfluxDBClient
import pandas as pd

client = InfluxDBClient(url="http://localhost:8086", token="my-token", org="my-org")
query = 'from(bucket:"tor_vpn_data") |> range(start:-1h) |> filter(fn:(r) => r._measurement == "tor_visits")'
df = client.query_api().query_data_frame(query)

# Flag sessions shorter than 1 minute
df['session_duration'] = df['_time'].diff().dt.total_seconds()
suspicious = df[df['session_duration'] < 60]
print(suspicious[['_time', 'url', 'session_duration']])

The results were displayed in Grafana, making it clear and visual for analysis.
