Shadow Tracker: Analyzing User Behavior in Tor and VPN

Shadow Tracker is a cybersecurity project focused on analyzing user behavior in privacy-centric environments like Tor and VPNs. By examining digital artifacts such as Tor browser cache, VPN logs, and memory dumps, it reconstructs user activities and identifies suspicious patterns, such as rapid IP changes or short browsing sessions.

This project simulates a digital forensics investigation, providing insights into how users interact with privacy tools. It is relevant for cybercrime investigations and threat intelligence, showcasing skills in cybersecurity and data analysis.

Technologies Used

  • Python: Parsing artifacts and analyzing data.
  • SQLite3: Extracting browsing history from Tor’s places.sqlite.
  • Volatility3: Analyzing memory dumps for Tor and VPN processes.
  • Pandas: Aggregating and processing artifact data.
  • Matplotlib: Creating simple visualizations like timelines.

These tools were selected for their reliability in digital forensics and data processing.

1. Setting Up the Environment

A virtual machine with Windows 10 and Ubuntu 20.04 was configured to simulate user activities in Tor and VPN environments.

  • Tor Browser: Installed to generate browsing artifacts.
  • OpenVPN: Set up with logging enabled to track VPN connections.
  • Volatility3: Configured for memory analysis.
  • Python: Used with libraries like pandas and sqlite3.

Tor Browser and OpenVPN were launched, and a memory dump was created to capture system state.

2. Collecting Artifacts

Artifacts from Tor and VPN usage were collected to analyze user behavior.

  • Tor Browser Cache: Parsed places.sqlite to extract URLs and timestamps.
  • VPN Logs: Analyzed OpenVPN logs for connection times and IP addresses.
  • Memory Analysis: Used Volatility3 to identify Tor and VPN processes.

3. Analyzing Behavior

User behavior was analyzed by examining browsing patterns and VPN usage. Key metrics included session duration, frequency of IP changes, and visited domains.

A simple timeline was created to visualize when users were active, highlighting unusual patterns like short Tor sessions.

4. Detecting Suspicious Activity

Suspicious behaviors were flagged using simple rules, such as sessions shorter than 1 minute or multiple IP changes in a short period.

The results were summarized in a table, making it easy to identify potential issues.

5. Troubleshooting

Several issues were encountered and resolved during the project:

6. Why It Matters

Shadow Tracker demonstrates how to analyze user behavior in privacy tools like Tor and VPNs, with applications in:

  • Cybercrime Investigations: Tracking activities in privacy-centric environments.
  • Threat Intelligence: Identifying suspicious patterns.

Future enhancements could include real-time monitoring or support for other privacy tools.
