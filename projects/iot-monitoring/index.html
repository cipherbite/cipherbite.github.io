IoT Data Monitoring System

This project is an IoT data monitoring system that tracks sensor information in real time. It uses technologies like MQTT, InfluxDB, Grafana, and Jupyter Notebook to collect, store, visualize, and analyze data. Here, it simulates temperature data, sending it via MQTT, saving it in InfluxDB, displaying it live in Grafana, and analyzing it in Jupyter Notebook. This kind of system could be useful in industry, smart homes, or precision agriculture, where up-to-date data matters.

1. Setting Up the Tools

I started by preparing the environment on Windows using WSL. I chose a set of free, widely-used tools that work well for IoT projects:

  • Mosquitto MQTT: A message broker running on port 1883 for device communication.
  • Node-RED: A tool for connecting system parts with a visual interface.
  • InfluxDB: A database for storing time-series data, like sensor readings.
  • Grafana: A platform for creating live data charts and visualizations.
  • Jupyter Notebook: An environment for data analysis and predictive modeling.

Each tool handles a specific part of the data process.

2. Database Configuration

I set up InfluxDB to store sensor data. I created a database with the command CREATE DATABASE iot_data and checked it with SHOW DATABASES. InfluxDB works well for time-series data, making it a good fit for this project.

3. Node-RED – Connecting the Parts

In Node-RED, I built a data flow that takes messages from MQTT, converts them to JSON, and saves them in InfluxDB. Its simple interface makes it easy to adjust the system.

4. Data Generator

I wrote a Python script in Jupyter Notebook to simulate a temperature sensor. It generates random values between 20–35°C every 2 seconds and sends them via MQTT. This helps test the system without physical hardware.

5. Data Verification

To check if data reached InfluxDB, I wrote a script to pull it out. The result was a table with temperatures and timestamps, showing everything worked as expected.

6. Visualization in Grafana

In Grafana, I set up a dashboard to show temperature data in real time. The charts make it easy to see changes and trends.

7. Running the System

After setup, I started Mosquitto, InfluxDB, Node-RED, and Grafana together. Data moved smoothly from the generator to the dashboard, confirming the system functioned correctly.

8. Data Analysis

In Jupyter Notebook, I analyzed the data, calculating stats like averages and creating charts, such as lines or histograms. This helps understand what’s happening with the data.

9. Prediction

I added a basic predictive model in Jupyter Notebook to estimate future temperature values based on past data. This could help spot issues early.
