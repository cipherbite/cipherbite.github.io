Solana Insiders Trading Analysis

This project investigates insider trading patterns on the Solana blockchain by monitoring wallet transactions in real time. It leverages the Solana API, SQLite, Streamlit, and NetworkX to collect, store, visualize, and analyze blockchain data efficiently.

The system tracks 10 predefined wallets, builds a network of SOL transfers, and provides insights through a dashboard. It aims to detect anomalies and correlations with SOL price movements, offering a foundation for blockchain forensics.

Potential applications include financial auditing, market manipulation detection, and blockchain security research.

Technologies Used

  • Solana API: Source of blockchain transaction data.
  • SQLite: Lightweight database for storing transactions and prices.
  • Streamlit: Interactive dashboard for data visualization.
  • NetworkX: Tool for building and analyzing transfer networks.
  • Pandas: Data manipulation and analysis.

These tools were selected for their efficiency, Python compatibility, and suitability for blockchain data analysis.

1. Setting Up the Environment

The project was configured on a Linux environment (WSL on Windows), with the following setup:

  • Python Virtual Environment: Isolated environment via venv.
  • Solana API: Public endpoint at https://api.mainnet-beta.solana.com.
  • SQLite: Local database in data/solana_data.db.
  • Streamlit: Web-based dashboard for real-time insights.
  • NetworkX: Graph analysis of wallet interactions.

2. Database Configuration

SQLite was set up to store transaction and price data with tables transactions and prices, created dynamically in the script.

3. Data Collection

A Python script fetches transaction signatures and details from the Solana API for 10 wallets, calculates net SOL changes, and builds a transfer graph. It respects API limits with a 6-second delay.

4. Network Analysis

NetworkX constructs a directed graph of SOL transfers, saved as solana_transfers.gml. This reveals connections between monitored wallets and others.

5. Visualization in Streamlit

A Streamlit dashboard displays SOL price trends, transaction details, and anomaly alerts. It includes filters for date and wallet, enhancing interactivity.

6. Running the System

The data collection script and dashboard run independently. Data flows from the Solana API to SQLite, then to Streamlit, validated by logs (e.g., 50 transactions on 2025-03-13).

7. Data Analysis

Pandas queries SQLite to analyze transaction volumes and correlate them with SOL price changes, identifying potential insider trading signals.

8. Troubleshooting
